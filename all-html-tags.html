Теги выделения и форматирования текста

<abbr title="..."> — аббревиатура

Используется для выделения аббревиатур в тексте. Позволяет дать им расшифровку в атрибуте title.
Пример: Принцип KISS должен соблюдаться везде, где только можно.

<em> — эмоциональное или смысловое выделение текста (emphasized)

Используется, когда требуется указать в тексте особую часть, на которую сделать отдельный акцент.
Пример: Нужно просто запомнить все основные теги.

<i> — выделение курсивом (italic \ idiomatic)

До HTML5 — использовался только для визуального выделения текста курсивом, не имел семантической нагрузки.

В HTML5 — используется для выделения части текста, которая требует альтернативного голоса или интонации. Так же применяется для: топонимов, технических терминов, иностранных слов, идиом и т.п.
Пример: мы просто закапывались в этой работе.
Пример: г.Москва — столица нашей Родины. Центральный процессор — важная деталь любого компьютера.

<strong> — важный текст

Используется для выделения важной\критически важной части в тексте. В отличие от <em> не несет эмоциональной или нюансовой нагрузки.
Пример: Переход улицы на красный сигнал светофора строго запрещен.

<b> — выделение жирным (bold \ bring attention)

Используется для привлечения внимания к части текста, без добавления акцента или важности. Применяется для ключевых слов, упоминании названия продукта в тексте и т.п.
Пример: Компания Apple планирует представить свой новый iPhone на своей ежегодной выставке.

<u> — выделение текста без привязки к его смысловой нагрузке и содержанию (unarticulated)

Использутеся в случаях, когда нужно выделить часть текста, не основываясь на его значении. Например, это может быть опечатка или особенное написание.
Пример: Он сообщил, что простыл и хочет надеть нозки.

<mark> — выделение фона текста (будто маркером)

Используется для выделения текста, когда требуется обозначить релевантные части (например, соответствующие поисковому запросу).
Пример: Люди часто спрашивают, как установить Linux. А потом удивляются.

<del> <ins> — перечеркнутый текст и новый текст (deleted \ inserted)

Позволяет показать правки, сделанные в тексте. В атрибуте cite можно указать ссылку на причину изменения, а в атрибуте datetime время, когда изменение было совершено.
Пример: Я знаю некоторые все теги.

<s> — перечеркнутый текст (strikethrough)

Используется для вычеркивания частей текста без добавления семантики "исправления".
Пример: Планы на день: прибраться в комнате, сходить в магазин.

<sup> <sub> — надстрочный и подстрочный текст (superscript \ subscript)

Используется строго в типографических нуждах, когда требуется написать формулу или дать сноску. Нельзя использовать для художественного оформления текста.
Пример <sup>: x2+y2=R2 Пример <sub>: C2H5OH

<small> — второстепенный комментарий к основному тексту

Используется для текста копирайта, правовой и уточняющей технической информации. Степень семантичности неопределенная.
Пример: сайт разработан в приложении "Блокнот"

<cite> — источник цитаты (citation)

Помечает текст, как источник цитирования. Может содержать ссылку <a> на источник.
Пример: "Кто не работает — тот ест" (Операция "Ы" и другие приключения Шурика)

<q cite="..."> — цитата (quotation)

Используется для выделения части текста в качестве цитаты.
Пример: Как говорит наш дорогой шев: Куй железо, не отходя от кассы!

<dfn> — определение (definition)

Используется для выделения термина, которому будет дано определение в содержащем его блоке. Блоками могут выступать p, dt, dd, section. Блоком, раскрывающим определение будет являться ближайший предок dfn.

<var> — выделение переменной (variable)

Используется для выделения переменной в тексте.
Пример: переменная status определяет дальнейшее поведение алгоритма.

<code> — выделение кода

Используется для выделения кода в тексте.
Пример: Тег <html> является основным тегом.

<kbd> — пользовательский ввод (keyboard input)

Помечает текст, как текст, который пользователь может ввести с клавиатуры (или иным способом).
Пример: Для получения подсказки введите --help;

<samp> — пример программного вывода (sample output)

Используется в случаях, когда текст представляет собой текстовое сообщение какой-либо программы (например, консольный вывод).
Пример: После завершения работы программы в консоли вы увидите: Press any key....

<output> — место вывода вычисляемых (скриптом) данных

Как правило используется в сочетании с формой, т.к. элемент имеет дополнительные атрибуты, связывающие его с какой-либо формой и ее данными:
for — список id элементов (через пробел), чьи значения используются в вычислении.
form — id формы, к которой относится вычисляемое значение. Если output находится внутри формы, то можно не указывать.
name — имя элемента, которое можно использовать для присваивания элементу вычисленного значения с использованием from.elements API.

Семантические блоки

<header> — заголовок

Предназначен для размещения внутри себя основной шапки сайта, а так же шапки article \ section. Может присутствовать на странице во множественном числе.

<main> — блок основного контента

Предназначен для размещения внутри себя основного содержимого страницы. Должен присутствовать на странице в единственном числе. Точнее — может присутствовать на странице многократно, но все дополнительные блоки main должны иметь атрибут hidden.

<nav> — блок навигации

Может присутствовать на странице многократно. Обычно оборачивается вокруг ul. Внутрь nav следует помещать только главные навигационные блоки. Отдельные ссылки или меню ссылок второстепенного значения можно не помещать внуть nav.

<aside> — блок второстепенного контента

Предназначен для размещения внутри себя второстепенного содержимого текущего семантического блока. Может присутствовать на странице во множественном числе.

<footer> — подвал

Предназначен для размещения внутри себя подвала страницы, а так же футеров article \ section.

<article> — блок статьи

Независимая часть страницы, которая может быть использована вне контекста страницы. Может быть вложен в другой элемент article или section.
Примеры использования: статья, запись в ленте новостей, виджет в боковой колонке.

<section> — блок секции

Составляющая часть какой-либо семантической сущности (например, секция статьи или отзыва). Может быть вложен в другой элемент section или article.

<article> VS <section>

article используется в случае, когда требуется определить одну и более независимых друг от друга (но семантически эквивалентых друг другу) семантических структур внутри какой-либо другой семантической структуры (body, article или section). При этом эти независимые структуры не являются свойством родительской структуры, а являются ее содержанием.
section используется в случае, когда для какой-либо структуры (body, article или section) нужно определить вложенные подструктуры, которые бы описывали какую-то часть родительской структуры.

Коротко:
article не является непосредственным свойством родительской структуры, а section является непосредственным свойством родительской структуры.

Иллюстрация структуры использования в псевдокоде:

-body
    -h1 Раздел "Статьи"
    -article
        -h1 статья 1...
    -article
        -h1 статья 2...
    -article
        -h1 статья 3...
    -section
        -h1 О чем этот раздел
    -section
        -h1 Подразделы раздела "Статьи"

<address> — блок адреса

Используется для указания адреса автора, фирмы и т.п. Может находиться внутри article, footer и других местах. Должен содержать только информацию об адресе. Внутри рекомендуется написать чей именно это адрес. Внутри может быть текст, ссылки и любые другие элементы, относящиеся к адресу.

Автор страницы: jenekomail@gmail.com
Адрес производства: Россия, д.Купчино.

<blockquote cite="..."> — блок цитаты (block quotation)

Используется в случае, когда весь текстовый блок является цитатой. Позволяет указать источник цитаты в атрибуте cite.

Пример:
Lorem ipsum dolor sit amet consectetur adipisicing elit. Assumenda excepturi repudiandae quam nihil repellat at nostrum cupiditate dolores facilis non nobis tenetur, tempora dignissimos necessitatibus eius libero odit. Alias, delectus?

<figure> и <figcaption> — контейнер информации c подписью к ней

figure может содержать изображение, диаграмму, фрагмент кода <code>, стихи, цитату blockquote и т.п. и подпись <figcaption>.

<details> и <summary> — раскрывающийся текст и его заголовок (нативный аккордеон)

Внутри может содержатья элемент summary, который станет заголовком элемента. Необязательный атрибут open определяет раскрыт текст или закрыт.

Краткое описание 1 Lorem ipsum dolor sit, amet consectetur adipisicing elit. Reprehenderit voluptatibus aspernatur blanditiis debitis pariatur ducimus, mollitia in nisi quae architecto dolorem consequuntur provident maiores officia fugiat recusandae aliquid voluptate natus?
Краткое описание 2 Lorem ipsum dolor sit, amet consectetur adipisicing elit. Reprehenderit voluptatibus aspernatur blanditiis debitis pariatur ducimus, mollitia in nisi quae architecto dolorem consequuntur provident maiores officia fugiat recusandae aliquid voluptate natus?

<data> — связывает свое содержимое со значением своего атрибута value

Используется в случае, когда для каких-то данных (например, модели телефона) указать технические данные, предназначенные для последующей обработки через JS. Если речь идет о данных времени, следует использовать элемент time.
Пример: мой планшет модели Wacom Intuos 4

<time> — связывает свое содержимое со значением своего атрибута datetime

Используется в случае, когда для каких-то данных времени требуется указать технические данные времени, предназначенные для последующей обработки через JS.
Пример: встреча состоится первого апреля в полдень.

Так же может использоваться без атрибута datetime.
Пример: перерыв на обед в 14-00.

Семантические блоки мультимедии

<picture> — контейнер изображений

Позволяет указать внутри себя несколько вариантов изображений, из которых браузером будет выбрано самое подходящее. Внутри должен присутствовать тег img с изображением по-умолчанию. Дополнительные изображения задаются элементами source. 

                <picture>
                    <source srcset="images/picture_img.webp">
                    <img src="images/picture_img.jpg" alt="">
                </picture>

<video src="..."> — видео

Источник видео может быть задан через атрибут src или через элемент source. Можно добавить несколько source с разными форматами видео (формат указывается в аргументе type). В этом случае будет выбран первый поддерживаемый формат.

Атрибуты

  • autoplay — автоматическое начало воспроизведения.
  • controls — отображать элементы управления.
  • src — источник медиа-ресурса. Можно использовать вместо source.
  • width и height — ширина и высота в px.
  • loop — зациклить.
  • muted — без звука.
  • playsinline — просит мобильные браузеры не переходить автоматически в полноэкранный режим при начале воспроизведения видео.
  • poster — URL изображения, показывающегося до начала воспроизведения видео.
  • preload — настройка загрузки: none (не предзагружать), metadata (предзагружать мета-данные), auto (на усмотрение браузера. Является значением по-умолчанию).

<audio src="..."> — аудио

Источник аудио может быть задан через атрибут src или через элемент source. Можно добавить несколько source с разными форматами аудио (формат указывается в аргументе type). В этом случае будет выбран первый поддерживаемый формат.

Атрибуты

  • autoplay — автоматическое начало воспроизведения.
  • controls — отображать элементы управления.
  • src — источник медиа-ресурса. Можно использовать вместо source.
  • loop — зациклить.
  • muted — без звука.
  • preload — настройка загрузки: none (не предзагружать), metadata (предзагружать мета-данные), auto (на усмотрение браузера. Является значением по-умолчанию).

<source> — источник

Задает медиа-ресурс, который будет отображен в блоке <picture>, <audio> или <video>. Выбор того или иного <source> определяется в зависимости от значений атрибутов.

Общие атрибуты

  • type — MIME-тип ресурса (опционально). Если не указан, то будет взят из загруженного файла.

Атрибуты для <picture>:

  • sizes — список (через запятую) из медиавыражения и желаемого размера изображения:
    sizes="(max-width: 320px) 400px, (min-width: 600px) 100vw"
  • srcset — список (через запятую) из ширины (или плотности) и соответствующего ей изображения:
    srcset="img2x.png 2x, img.png 100w"
  • media — медиавыражение, при котором будет выводиться ресурс:
    media="(max-width: 400px) 100vw, 720px"

Атрибуты для <audio> и <video>:

  • src — источник медиа-ресурса.

<track> — дополнительные текстовые данные для audio и video

Позволяет указать для ресурса синхронизированные текстовые дорожки (например, субтитры).

<hgroup> — группа заголовков

Позволяет сгруппировать заголовки, например h1 и h2 так, что второй заголовок будет считаться подзаголовком первого. Не рекомендуется к применению из-за проблем с семантичностью.

Контент и прочее

<hr> — разделитель (Horizontal Rule)

<noscript> — контент при отключенном JS

<img src="..."> — изображение (image)

<svg> — векторная svg-графика (scalable vector graphic)

Может быть вставлена как через элемент <svg>, так и в качестве значения атрибута src элемента img.

<map> и <area> — карта изображения и ее области

Позволяет указать для изображения кликабельные области. Каждая область описывается отдельным элементом area. Для присвоения map к img у map треубется указать атрибут name, после чего у img указать атрибут usemap равным #имя_map.

<iframe src="..."> — встроенное содержимое

<object type="..." data="..."> — элемент внешнего ресурса

Позволяет вставить pdf, flash и т.п. ресурсы, которые будут обработаны плагином браузера. Атрибуты type (MIME-тип ресурса) и data (URL ресурса) являются обязательными.

<param> — параметры, передаваемые в ресурс элемента object

Позволяет через атрибуты name и value передать произвольные параметры в ресурс, подключаемый через object. Может быть больше одного.

<embed type="..." src="..."> — элемент внешнего ресурса

Позволяет вставить mp4 и т.п. ресурсы, которые будут обработаны плагином браузера. Атрибуты type (MIME-тип ресурса) и src (URL ресурса) являются обязательными. По-сути повторяет функционал object.

<template> — контент для последующего отображения средствами JS

Содержимое будет обработано парсером на валидность, но не будет отображено на странице. Используется в случаях, когда требуется подготовить какую-либо структуру, которая впоследствии, силами JS, может быть дозаполнена данными, после чего отображена в документе.

Обратиться к содержимому template через JS можно через специальное свойство content. Внутри содержится объект DocumentFragment, который требуется клонировать методом cloneNode(true), после чего клон может быть вставлен в документ.
Не рекомендуется вставлять исходный объект DocumentFragment. Подробнее см. Вебкомпоненты (webComponents).

<slot> — место в вебкомпоненте, куда может быть передано произвольное содержимое (текст или разметка)

Подробнее см. Вебкомпоненты (webComponents).

<canvas> — элемент рисования графики и анимации

<progress> — прогресс-бар

Используется для отображения степени завершенности процесса. Текущее значение задается атрибутом value, конечное значение задается атрибутом max.

<meter> — измерительная шкала

Используется для визуального отображения значения, находящегося в каких-либо рамках. Рамки задаются атрибутами min, low, optimum, high и max. Текущее значение шкалы задается атрибутом value. В зависимости от того, в каких рамках находится текущее значение, шкала будет иметь определенный цвет.
Значение шкалы можно привязать к какой-либо форме. Для этого используется атрибут form="id-формы".

<dialog> — аналог модального окна

Имеет атрибут open, наличие или отсутствие которого определяет, будет ли показано содержимое элемента.
Не поддерживается IE, Firefox, Safari \ iOS

Текст и элементы текста

<li> — элемент списка (list item)

<dl> <dt> и <dd> — список описаний (description list)

Используется для списков, элементы которого представляют из себя термин dt (description term) и его описание dd (description details).

Табличные теги

Место отображения (сверху или снизу) может быть задано через CSS-свойство caption-side: [top|bottom]

<colgroup> — определяет стили одного и более столбцов таблицы (group of columns)

Размещается на том же уровне, что и tr элементы после опционального заголовка caption. Можно не использовать закрывающий тег, если внутри не используются col. Так же, в этом случае можно задать атрибут span, который определяет количество столбцов, для которых будут определены стили.
Список доступных CSS-стилей ограничен: можно задать border, background, width и visibility.

<col> — определяет стиль отдельного столбца таблицы. Используется внутри colgroup

Размещается внутри colgroup, определяет стиль столбца. Так же, как и colgroup может содержать атрибут span, который определяет количество столбцов, для которых определяются стили.
Список доступных CSS-стилей ограничен: можно задать border, background, width и visibility.

<optgroup label="..."> — группировка опций select

Позволяет сгруппировать опции внутри select и добавить каждой группе заголовок через атрибут label.

<datalist> — список вариантов значений инпута

Позволяет внутри себя перечислить варианты значений для input. Каждый вариант определяется элементом code{<option value="...">}, значение которого указывается в атрибуте value. При этом option используется без содержимого и закрывающего тега.
Чтобы привязать datalist к какому-либо input, у этого input требуется указать атрибут list="..." содержащий ID нужного списка.
В отличие от select, datalist не ограничивает значение input указанным списком. Если требуемое значение в списке отсутствует, пользователь все равно имеет возможность вписать его вручную.
